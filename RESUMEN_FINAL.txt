🚀 CÓMO EJECUTAR
═══════════════════════════════════════════════════════════════════════════════

  1. Abre una terminal en el directorio del proyecto
  
  2. Instala dependencias (si no están ya instaladas):
     $ npm install
  
  3. Inicia el servidor:
     $ npm start
  
     O en modo desarrollo con auto-reload:
     $ npm run dev
  
  4. El servidor estará disponible en:
     🔗 http://localhost:3000

⚙️ API REST (Endpoints disponibles)
═══════════════════════════════════════════════════════════════════════════════

  GET    /api/products              → Obtener todos los productos
  GET    /api/products?limit=5      → Obtener con límite
  GET    /api/products/:id          → Obtener un producto específico
  POST   /api/products              → Crear nuevo producto
  PATCH  /api/products/:id          → Actualizar producto
  DELETE /api/products/:id          → Eliminar producto

🔄 CÓMO FUNCIONA EL FLUJO EN TIEMPO REAL
═══════════════════════════════════════════════════════════════════════════════

  Usuario A abre realTimeProducts
       ↓
  Socket se conecta automáticamente
       ↓
  Servidor envía productos iniciales
       ↓
  Usuario A crea un producto
       ↓
  socket.emit('createProduct', data)
       ↓
  ProductManager.addProduct()
       ↓
  io.emit('productCreated', product)  ← Se emite a TODOS los clientes
       ↓
  ¡ Usuario A y Usuario B ven el nuevo producto automáticamente !
       ↓
  Los cambios se persisten en src/db/products.json
